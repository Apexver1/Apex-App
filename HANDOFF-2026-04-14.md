### Key gotchas (burn in)
1. Supabase SQL editor runs ONLY the first statement. Paste one statement at a time, OR wrap in a DO block, OR use a WITH CTE chain.
2. `confirmed_at` on `auth.users` is now GENERATED — remove from any INSERT/UPDATE.
3. Supabase default API row cap is 1000. Paginate `.select()` for full-table queries.
4. Password min length in Supabase appears to be 8 (rejected `demo123`, accepted `demo2026`).
5. Direct `auth.users` INSERT without matching `auth.identities` row may silently break sign-in.
6. BallDontLie `/players?team_ids[]=X` returns ALL-TIME historical players, not current roster. Height/weight are NULL.
7. Safari tab cache: hard-reload with **Cmd+Shift+R** after GitHub Pages redeploys. Sometimes needs two reloads.
